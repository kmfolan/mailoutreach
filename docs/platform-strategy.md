# Outbound Forge Platform Strategy

## Current Goal

Ship a protected web MVP that can be used internally or by a small group of operators.

## Current Platform

- web app served from the Node server
- local launcher flow for development
- next deployment target: DigitalOcean

## Near-Term Delivery Order

1. stable local web experience
2. protected public web deployment
3. real database
4. integration layer for provisioning

## Longer-Term Expansion

If the product grows into a multi-user SaaS or operations tool, consider:

- dedicated front-end app structure
- role-based admin access
- team accounts
- audit history
- workflow job tracking

## What Not To Build Yet

Until the planning-vs-provisioning boundary is locked:

- mobile apps
- desktop wrappers
- broad cross-platform packaging

The fastest route is still a strong web product first.
